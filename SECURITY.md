# Security Policy

## Training Repository Notice

This repository is intentionally vulnerable for local training and controlled demos. It is not a production system and must never be exposed to the public internet.

## Supported Use

- Local lab work on systems you control
- Private learning and demonstrations
- Documentation, workflow, and reproducibility improvements

## Not a Real Bug Bounty Target

Do not treat this repository or any deployment of it as a public bug bounty program. Do not test third-party systems using these materials unless you have explicit written authorization.

## How To Report Repository Problems

If you find a problem with the repository itself:

- Use GitHub Issues for reproducible bugs in docs, scripts, or workflows
- Use GitHub Discussions for questions, ideas, and learning-oriented conversations

## Safe Disclosure Expectations

When reporting a problem, please avoid including:

- secrets or tokens
- real target data
- unsafe internet-facing deployment guidance

Share only the minimum evidence needed to reproduce the issue in a local training environment.

## Security Improvements Welcome

Helpful contributions include:

- safer defaults for local use
- clearer warnings and setup guidance
- reproducible validation steps
- better contributor and support documentation
